# バランス状況レポート

シミュレーションの現状をNPB実データと比較し、わかりやすく報告するスキル。

## 使い方

- `/status` — 最新のバランステスト結果（200試合）を実行して全指標を報告
- `/status quick` — 直近のテスト結果ファイルがあればそれを表示（なければ100試合テスト実行）
- `/status full` — 500試合テストを実行して詳細レポート

## やること

### 1. バランステスト実行

引数に応じて試合数を決定:
- `quick`: 100試合
- (引数なし): 200試合
- `full`: 500試合（`NODE_OPTIONS="--max-old-space-size=8192"` 付き）

```bash
npx tsx scripts/test-balance-full.ts --games=<N>
```

### 2. NPB平均との比較レポートを作成

以下のフォーマットで報告する:


### 3. NPBベンチマーク値（レポート作成時に参照）

打撃:
- チーム打率: .260前後（首位打者.330+）
- BABIP: .290-.310
- K%: 18-22%
- BB%: 7-10%
- HR/試合: NPB平均1.0-1.2本
- GO/AO: 1.0-1.3

打球タイプ別ヒット率:
- ゴロ: 23-25%
- ライナー: 50-55%（最もヒットになりやすい）
- フライ: 15-20%
- ポップ: ~5%

### 4. 乖離の自動検出ルール

以下の場合に⚠️を付ける:
- 目標範囲を外れている指標
- NPB平均から±30%以上乖離している指標
- 品質ゲート（OF回収率>75%, P回収率<3%等）を未達の項目
