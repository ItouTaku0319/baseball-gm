# タスク委譲

引数で指定されたタスクを適切なエージェントに委譲する。

## 手順

1. タスク内容を分析して適切なエージェントを選定:
   - `engine/` 関連 → `impl-engine`
   - `app/` `components/` 関連 → `impl-ui`
   - `models/` `store/` `db/` 関連 → `impl-data`
   - 複数領域 or 小規模 → `implementer`

2. タスクに完了条件を付与:
   - `npm run build` 成功（必須）
   - テストがあれば通過（必須）
   - バランステスト（シミュレーション変更時）

3. Task ツールでエージェントを起動:
   - 関連ドキュメントへの参照を含める
   - 完了条件を明記
   - 変更理由と変更前後の値を報告するよう指示

4. 結果をユーザーに報告:
   - 変更されたファイル一覧
   - テスト結果
   - 注意事項
