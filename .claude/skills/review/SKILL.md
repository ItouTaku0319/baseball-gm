# 変更レビュー

直近のコミットまたは未コミットの変更を素早くレビューする。

## 手順

1. `git status` と `git diff` で変更内容を把握
2. 変更されたファイルを読み、以下を確認:
   - CLAUDE.md のコーディング規約に違反していないか
   - 型にフィールド追加時は optional (`?:`) か
   - マジックナンバーが `physics-constants.ts` に定数化されているか
   - 不要なコメント・docstring が追加されていないか
3. `npm run build` が通るか確認
4. シミュレーション変更があれば `npm run test` の結果を確認
5. 問題があれば具体的な修正案を提示、なければ「LGTM」

## 出力フォーマット

```
## レビュー結果: [LGTM / 要修正]

### 変更サマリ
- 変更ファイル数: N
- 追加/削除行数: +X / -Y

### 確認項目
- [ ] ビルド通過
- [ ] テスト通過
- [ ] 規約準拠

### 指摘事項（あれば）
1. ...
```
