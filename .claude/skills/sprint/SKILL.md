# スプリント管理

`task.md` の「現在のスプリント」セクションを読み、状態を報告する。

## 使い方

- `/sprint` — 現在のスプリント状況を表示
- `/sprint plan` — 次のスプリントを計画する（バックログから選定）
- `/sprint done` — スプリント振り返り + 次の計画

## やること

### `/sprint` (引数なし)
1. `task.md` の `## 現在のスプリント` セクションを読む
2. 各タスクの完了状況を確認（git log でコミット有無、テスト通過状況）
3. 以下を報告:
   - スプリント名と期間
   - タスク一覧と進捗（未着手/進行中/完了）
   - ブロッカーがあれば警告
   - 次にやるべきアクション

### `/sprint plan`
1. `task.md` のバックログを読む
2. `docs/project-map.md` の成熟度を参照
3. 2-3日で完了できるタスクを3-5個提案
4. ユーザーの承認を得て `task.md` を更新

### `/sprint done`
1. 完了したタスクをまとめる
2. git log から今スプリントのコミットを列挙
3. 未完了タスクの理由と次スプリントへの持ち越し判断
4. `/sprint plan` を続けて実行
