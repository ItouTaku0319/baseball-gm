# 守備台本（Play Script）設計書

## 概要

打球発生時、打球の物理特性とゲーム状況から**台本（Play Script）**を生成し、
9人の野手それぞれに「役割」を割り当てる。野手はその役割に従って物理的に移動する。

**設計思想**: 「結果の正しさ」と「動きの自然さ」を分離して保証する
- 結果（アウト/ヒット）: 台本生成時に確率判定で決定
- 動き: 各野手が割り当てられた役割を物理シミュレーションで実行

---

## 1. 台本の構造

### 1.1 台本生成の入力

```typescript
interface ScriptInput {
  // 打球情報
  ballType: 'ground' | 'line_drive' | 'fly' | 'popup';
  direction: number;        // 角度（0=センター、-45=三塁線、+45=一塁線）
  exitVelocity: number;     // 打球速度 km/h
  launchAngle: number;      // 打ち出し角度
  landingPos: Vec2;         // 着地予測位置
  landingTime: number;      // 着地予測時間

  // ゲーム状況
  runners: RunnerSituation; // どの塁にランナーがいるか
  outs: 0 | 1 | 2;
  inning: number;
  scoreDiff: number;        // 守備側から見た点差（+はリード）

  // 野手情報
  fielders: FielderInfo[];  // 9人の位置・能力値
}
```

### 1.2 台本の出力

```typescript
interface PlayScript {
  // 結果（事前に決定）
  outcome: PlayOutcome;      // アウト種別 or ヒット種別
  catcherId: FielderPosition; // 誰が捕球するか

  // 9人の役割
  assignments: Map<FielderPosition, FielderRole>;

  // 送球計画（捕球後）
  throwPlan: ThrowPlan;
}

// 野手に割り当てられる役割
type FielderRole =
  | { type: 'pursue'; target: Vec2 }           // 打球を追う（捕球担当）
  | { type: 'backup_pursue'; target: Vec2 }    // 追球のバックアップ
  | { type: 'cover_base'; base: Base }          // ベースに入る
  | { type: 'cutoff'; position: Vec2 }          // 中継位置に移動
  | { type: 'backup_base'; base: Base }         // ベース後方のバックアップ
  | { type: 'backup_throw'; position: Vec2 }    // 送球ラインのバックアップ
  | { type: 'hold'; position: Vec2 }            // 定位置で待機
  | { type: 'trail_runner'; runner: Base }       // ランナーを追う（タグプレー用）
```

### 1.3 送球計画

```typescript
interface ThrowPlan {
  primary: {
    target: Base;           // 送球先
    urgency: 'force' | 'tag' | 'routine'; // フォースアウト/タグ/ルーティン
  };
  secondary?: {
    target: Base;           // ダブルプレー時の2送球目
  };
  condition?: string;       // 条件付き（例: "ランナーが走っていたら2塁"）
}
```

## 2. 方向ゾーンの定義

ホームベースから見た打球方向を9つのゾーンに分類する。
角度は三塁線=-45°、センター=0°、一塁線=+45°。

```
          CF(0°)
     LCF(-15°)  RCF(+15°)
   LF(-30°)        RF(+30°)
     3B(-40°)    1B(+40°)
       SS(-20°) 2B(+20°)
            P(0°)
          C(ファウル)
```

### 内野ゾーン（ゴロの処理者を決定）

| ゾーン | 角度範囲 | プライマリ | セカンダリ |
|--------|----------|-----------|-----------|
| Z1: 三塁線 | -45° ~ -35° | 3B | — |
| Z2: 三遊間 | -35° ~ -20° | SS or 3B | 近い方 |
| Z3: 遊撃正面 | -20° ~ -10° | SS | — |
| Z4: 二遊間 | -10° ~ +10° | SS or 2B | 近い方 |
| Z5: 二塁正面 | +10° ~ +20° | 2B | — |
| Z6: 一二塁間 | +20° ~ +35° | 2B or 1B | 近い方 |
| Z7: 一塁線 | +35° ~ +45° | 1B | — |
| Z8: 投手方向 | -10° ~ +10°, 短距離 | P | SS/2B |
| Z9: 捕手方向 | ファウルポップ | C | 1B/3B |

### 外野ゾーン（フライの処理者を決定）

| ゾーン | 角度範囲 | プライマリ | バックアップ |
|--------|----------|-----------|-------------|
| OF-L: レフト | -45° ~ -20° | LF | CF |
| OF-LC: 左中間 | -20° ~ -8° | CF or LF | もう一方 |
| OF-C: センター | -8° ~ +8° | CF | LF or RF |
| OF-RC: 右中間 | +8° ~ +20° | CF or RF | もう一方 |
| OF-R: ライト | +20° ~ +45° | RF | CF |

**優先ルール**: 同距離の場合、センターが優先（CF > LF/RF）。
外野手同士の間の打球は、**前進してくる方**が優先（後退より前進の方が捕球しやすい）。

## 3. ゴロ（Ground Ball）の台本

ゴロは内野手が処理するプレーの中核。台本のパターンが最も多い。

### 3.1 基本原則

- **捕球者の決定**: 方向ゾーン + 野手の守備範囲（fielding能力）で決まる
- **送球先**: ランナー状況とアウトカウントで決まる
- **ダブルプレー**: ランナー1塁 + 0-1アウト → 併殺体制
- **全員が動く**: 捕球者以外にも必ず役割がある

### 3.2 ランナーなし

最もシンプルなケース。捕球→一塁送球。

| 捕球者 | 1B | SS | 2B | 3B | P | LF | CF | RF |
|--------|----|----|----|----|---|----|----|-----|
| P | cover_1B | cover_2B | hold | hold | **pursue** | hold | backup | hold |
| 3B | cover_1B | cover_2B | cover_3B | **pursue** | backup_throw | hold | hold | hold |
| SS | cover_1B | **pursue** | cover_2B | cover_3B | backup_throw | hold | backup | hold |
| 2B | cover_1B | cover_2B | **pursue** | hold | backup_throw | hold | backup | hold |
| 1B | cover_1B* | cover_2B | hold | hold | **cover_1B** | hold | hold | backup |

*1Bが捕球した場合: 自分でベースを踏むか、Pがカバーに入る

**送球先**: 1B（ルーティン）

**外野手の動き**:
- 打球方向に最も近い外野手: バックアップ（打球方向に前進）
- 他の外野手: 定位置で待機

### 3.3 ランナー1塁（0-1アウト）— ダブルプレー体制

最も重要なパターンの一つ。プロ野球の守備の見せ場。

**基本フォーメーション**:
- 2B/SS: 打球方向によってどちらかがベースに入る
  - 左方向のゴロ(Z1-Z3) → **2Bが二塁ベースに入る**（ピボットマン）
  - 右方向のゴロ(Z5-Z7) → **SSが二塁ベースに入る**（ピボットマン）
  - センター方向(Z4) → 近い方が処理、もう一方がベースに入る
- 1B: 捕球後に一塁ベースに戻るか、Pがカバー

| 捕球者 | 送球1 | ピボット | 送球2 | 1Bカバー |
|--------|-------|---------|-------|---------|
| 3B | → 2B | 2B | → 1B | 1B or P |
| SS | → 2B | 2B | → 1B | 1B or P |
| 2B | → SS(2B) | SS | → 1B | 1B |
| 1B | → SS(2B) | SS | → 1B | P |
| P | → SS/2B | 近い方 | → 1B | 1B |

**ダブルプレーの判定条件**:
1. ゴロの速度が十分（遅いゴロはDP困難）
2. 捕球位置が二塁に近い（遠い場合は一塁のみ）
3. ランナーの走力（速いランナーは間に合わない）
4. 2アウト時はDPを狙わない → 一塁アウトのみ

**2アウト時**: ダブルプレー不要 → 一塁送球のみ

### 3.4 ランナー2塁（0-1アウト）

**目的**: 3塁への進塁を防ぎつつアウトを取る

| 打球方向 | 捕球者 | 送球先 | 備考 |
|---------|--------|--------|------|
| 左方向(Z1-Z3) | 3B/SS | 1B | ランナーは3塁へ進む（止められない） |
| 右方向(Z5-Z7) | 2B/1B | 1B | SSが三塁カバー |
| 投手方向(Z8) | P | 状況次第 | R2が飛び出していたら三塁送球 |

**カバー配置**:
- SS: 三塁カバー（左方向のゴロ以外）
- 3B: 三塁ベース待機（左方向のゴロで自分が捕球しなかった場合）

**2アウト時**: 全てのゴロで二塁ランナーは走る → 一塁でアウトを取るだけ

### 3.5 ランナー3塁（0アウト）

**最重要**: 得点を防ぐ
- 内野前進守備（ゲーム状況次第）
- 捕球後すぐホーム送球（ランナーが走っている場合）
- ランナーがスタートを切っていない → 一塁送球

| 状況 | 守備体制 | 送球先 |
|------|---------|--------|
| 0アウト, 僅差 | 前進守備 | ホームor一塁（ランナー判断で） |
| 0アウト, リード大 | 通常守備 | 一塁（得点許容） |
| 1アウト, 僅差 | 前進守備 | ホームor一塁 |
| 2アウト | 通常守備 | 一塁 |

**前進守備のポジション**:
- 内野手全員が約3m前に出る（ゴロを早く処理してホーム送球）
- ただしヒットゾーンが広がるリスクあり

### 3.6 ランナー1,2塁（0-1アウト）

**ダブルプレー体制 + 三塁カバー**

- 三塁はフォースプレー → 三塁送球→一塁転送のDP可
- 二塁経由のDPも可
- 打球方向で最適な併殺ルートが変わる

| 捕球者 | DP優先ルート | 代替 |
|--------|-------------|------|
| 3B | 3B→(自分で踏む)→1B | 3B→2B→1B |
| SS | SS→3B→1B | SS→2B→1B |
| 2B | 2B→SS(2B)→1B | 2B→3B(近ければ) |
| P | P→3B or 2B | 状況次第 |
| 1B | 1B→3B or 2B | — |

### 3.7 満塁

**全ベースがフォースプレー**

- 最も近いベースに送球するのが基本
- ホームゲッツー（本塁→一塁の併殺）が最高効率
- 捕球者が近い塁を踏んでから一塁へ転送

| 捕球者 | 最優先送球 | 備考 |
|--------|-----------|------|
| P | ホーム（C） | ホームゲッツー狙い |
| 3B | ホーム or 3B踏む | 距離次第 |
| SS | 2B or 3B | 近い方 |
| 2B | 2B踏む→1B | DP |
| 1B | 1B踏む→ホーム | DP |
| C | 本塁踏む→1B | ホームゲッツー |

### 3.8 ゴロの捕球結果判定

台本生成時に結果を確率で決定する:

```
捕球成功率 = base_rate
  × fieldingBonus(fielding_skill)    // 守備力
  × catchingBonus(catching_skill)    // 捕球力
  × velocityPenalty(exit_velocity)   // 打球速度によるペナルティ
  × reachPenalty(distance_to_ball)   // 打球到達距離によるペナルティ
```

**結果の分岐**:
1. **アウト** (捕球成功 + 送球成功)
2. **エラー** (捕球成功だが送球失敗、または捕球失敗)
3. **内野安打** (野手が到達できない、または間に合わない)
4. **外野への抜け球** (内野を完全に抜ける → 実質シングルorダブル)

## 4. フライ（Fly Ball）の台本

### 4.1 基本原則

- **捕球者の決定**: 外野ゾーン + 野手の足の速さ + 守備範囲で決まる
- **優先ルール**: 前進してくる野手 > 後退する野手、CF > コーナーOF
- **タッチアップ**: 0-1アウトで捕球された場合、ランナーはタグアップ可能
- **犠牲フライ**: 3塁ランナー + 0-1アウト + 十分な深さのフライ

### 4.2 外野フライ — 捕球者と結果

**捕球判定**:
```
到達可能 = (野手の移動可能距離 > 打球着地点までの距離)
移動可能距離 = (flightTime - reactionTime) × runSpeed × directionFactor
directionFactor = 0.85 (後退時) / 1.0 (横方向) / 1.05 (前進時)
```

**結果**:
- 到達可能 → 捕球成功（フライアウト） ※エラー率は低い（~2%）
- 到達不可能 → ヒット（落下地点と壁までの距離で単打〜三塁打）

### 4.3 外野フライ — 他の野手の役割

| 役割 | 担当者 | 動き |
|------|--------|------|
| バックアップ | 隣の外野手 | 捕球者の後方に回る |
| 中継 | SS or 2B | 送球ラインに位置取る |
| ベースカバー(2B) | SS or 2B | 中継に出なかった方 |
| ベースカバー(3B) | 3B | 三塁で待機 |
| ベースカバー(1B) | 1B | 一塁で待機 |
| ベースカバー(HOME) | C | ホームで待機 |
| 特になし | P | マウンド付近、バックアップ待機 |

**中継（カットオフ）の位置**:
- レフト方向 → SSが中継、2Bが二塁カバー
- センター方向 → SS or 2Bが中継（打球方向で判断）
- ライト方向 → 2Bが中継（一塁方向ライン上）、SSが二塁カバー

### 4.4 タグアップと犠牲フライ

**タグアップ条件**（ランナーの判断）:
1. 0-1アウトである（2アウトは強制走塁なのでタグアップ不要）
2. フライが捕球された
3. 次の塁まで間に合う見込みがある

**犠牲フライ成功率の要因**:
- フライの深さ（外野フェンス近く → 成功率高い、浅いフライ → 低い）
- 外野手の肩力（arm skill）
- ランナーの走力
- 外野手の捕球位置からの送球距離

**判定**:
```
送球時間 = 捕球後の動作(0.8-1.2s) + 距離 / 送球速度
ランナー時間 = 塁間距離(27.431m) / ランナー速度
タグアップ成功 = ランナー時間 < 送球時間
```

### 4.5 浅いフライ — 内野と外野の狭間

打球が内野後方〜外野前方に落ちるケース（テキサスヒット候補）。

**判定ポイント**:
- 内野手が後退して捕れるか？
- 外野手が前進して捕れるか？
- どちらも届かない場合 → テキサスリーグヒット

**優先ルール**: 外野手の前進 > 内野手の後退（前進の方が捕球しやすい）

## 5. ライナー（Line Drive）の台本

### 5.1 特徴

- 最も速い打球（反応時間が短い）
- アウト率は50-70%（NPB実績）
- 捕球できなかった場合、長打になりやすい

### 5.2 捕球判定

```
反応可能 = reactionTime(fielding_skill) < ボール到達時間
捕球圏内 = 野手位置からの横方向距離 < catchReach(fielding_skill)
```

**ライナーの特殊性**:
- 内野ライナー: 打球速度が速すぎて反応できないことが多い → 抜ける → ヒット
- 正面のライナー: 反応時間が十分 → ライナーアウト
- 外野ライナー: フライに近い処理（追球して捕球）

### 5.3 ライナー捕球後のダブルプレー

ライナーを捕球した場合、飛び出したランナーを戻す前にベースに送球→DP。
- 捕球者の近くのベースに送球（ランナーが飛び出している場合）
- 特に1塁ランナーは飛び出しやすい

## 6. ポップフライ（Popup）の台本

### 6.1 特徴

- 高く上がって内野に落ちる打球
- ほぼ確実にアウト（エラー率~1%）
- 風の影響を受けやすい

### 6.2 優先ルール（インフィールドフライ）

**重要**: 内野ポップフライの捕球優先順位

1. **内野手** > **捕手** > **投手**
2. 内野手の中では**ボールに近い者**が優先
3. 同距離の場合: **二遊間 > 一三塁 > 投手 > 捕手**
4. ファウルゾーンのポップ: **捕手 > 一塁手/三塁手**

**理由**: 内野手は前方を向いているので捕球しやすい。捕手は後方を向く必要がある。

### 6.3 インフィールドフライルール

- ランナー1,2塁 or 満塁 + 0-1アウト
- 内野フライ（審判の判断）
- 打者自動アウト → ランナーは塁にとどまる
- **台本での処理**: フラグを立てて、捕球成否にかかわらず打者アウト

### 6.4 他の野手の動き

ポップフライはほぼアウトなので、他の野手は最小限の動き:
- 捕球者: ボールの落下地点へ移動
- 近くの野手: 声をかけて譲る（ゲーム内では"離れる"動きで表現）
- 残りの野手: ベースカバー（走塁に備える）

## 7. 特殊プレーの台本

### 7.1 バント

- 投手・一塁手・三塁手がチャージ
- 二塁手が一塁カバー
- 遊撃手が二塁カバー
- ランナー状況で送球先が変わる

| ランナー | 送球優先先 | 備考 |
|---------|-----------|------|
| なし | 1B | 確実にアウト |
| 1塁 | 2B → 1B | 間に合えば2塁、無理なら1塁 |
| 2塁 | 3B → 1B | 三塁に行かせない |
| 1,2塁 | 3B → 2B → 1B | 状況次第 |

### 7.2 ヒットエンドラン

- ランナーが投球と同時にスタート
- 打者は必ず打つ → ゴロを打つことが多い
- 二遊間が空く（ランナーカバーでどちらかが動く）
- → 二遊間を抜けるヒットになりやすい

### 7.3 ダブルスチール

- 守備側のカバーが複雑になる
- 台本: 2塁送球が基本だが、3塁ランナーの飛び出しを狙う場合もある

## 8. 送球先決定のルールテーブル

### 8.1 基本原則

1. **フォースアウトを優先** — タグプレーより確実
2. **リードランナーを優先** — ただし確実に取れる場合のみ
3. **確実なアウト > 理想的なアウト** — 二兎を追わない
4. **2アウト時は確実に** — 最も近いベースで確実にアウト

### 8.2 ゴロの送球先テーブル

| ランナー | 0アウト | 1アウト | 2アウト |
|---------|--------|--------|--------|
| なし | 1B | 1B | 1B |
| 1B | 2B→1B (DP) | 2B→1B (DP) | 1B |
| 2B | 1B | 1B | 1B |
| 3B | HOME or 1B※ | HOME or 1B※ | 1B |
| 1B,2B | 3B→1B or 2B→1B | 近い塁→1B | 1B |
| 1B,3B | HOME→1B or 2B→1B | 2B→1B or 1B | 1B |
| 2B,3B | HOME or 1B※ | HOME or 1B※ | 1B |
| 満塁 | HOME→1B (DP) | HOME→1B | 最寄りフォース |

※ 前進守備の場合はHOME、通常守備は1B

### 8.3 送球成功率

```
送球成功率 = base_rate(0.95)
  × armBonus(arm_skill)          // 肩力
  × distancePenalty(throw_dist)   // 送球距離
  × rushPenalty(time_pressure)    // 急いだ送球の精度低下
  × pivotPenalty(is_dp_pivot)     // DP転送時のペナルティ
```

## 9. 結果判定のフロー

### 9.1 全体フロー

```
1. 打球の物理特性を計算（ball-trajectory.ts）
     ↓
2. 打球タイプと方向ゾーンを判定
     ↓
3. 捕球候補者を列挙（ゾーン→プライマリ/セカンダリ）
     ↓
4. 到達判定（野手の能力値 vs 打球の物理特性）
     ↓
5a. 到達可能 → 捕球/エラー判定 → アウト種別決定
5b. 到達不可能 → ヒット種別決定（落下地点の距離で）
     ↓
6. 台本生成（9人の役割を割り当て）
     ↓
7. 物理シミュレーション実行（ビジュアライズ用）
```

### 9.2 ヒット種別の決定

内野を抜けた打球の場合:

| 落下地点 | ランナーなし | ランナーあり |
|---------|------------|------------|
| 内野〜浅い外野 (40-60m) | シングル | シングル |
| 外野中間 (60-90m) | シングル | シングル/ダブル |
| 深い外野 (90-110m) | ダブル | ダブル/トリプル |
| フェンス際 (110m+) | ダブル/トリプル | トリプル |
| フェンス越え | ホームラン | ホームラン |

**補正要因**:
- 外野手の足の速さ → 処理が早ければ進塁を防げる
- 外野手の肩力 → 送球が速ければランナーを刺せる
- ランナーの走力 → 速いランナーは余分に進塁
- 打球の転がり方 → ゴロで抜けた場合はフライより処理が遅い

## 付録C: 台本方式と自律方式の比較

| 観点 | 自律方式（現在） | 台本方式（提案） |
|------|---------------|---------------|
| 知識の所在 | コード全体に分散 | テンプレートに集約 |
| デバッグ | 困難（相互作用） | 容易（台本単位） |
| 正確性 | 創発的（保証困難） | 宣言的（検証可能） |
| 拡張性 | 新パターン＝新ロジック | 新パターン＝新テンプレート |
| 動きの自然さ | 高い（自律的） | 中〜高（物理実行で補完） |
| パフォーマンス | 低い（毎tick判断） | 高い（事前決定） |
